# 以公平与高效为导向的MEV拍卖平台

本文提出以公平与高效为导向的MEV拍卖平台设计方案，并给出具体实现，即MP-RPC节点，通过引入分组聚合、价值排序、定时发送以及异常检测等机制，旨在在保证系统公平性、稳健性和效率的前提下，优化MEV拍卖流程。

仓库说明：

- `bsc-rpc`:本文提出的拍卖平台方法的具体实现，即MP-RPC的原型实现
- `mev-share-node-main`:Flashbots提出的MEV-Share Node方案的具体实现，注意这里是经过改造之后的版本（为了支持本章实验），如果需要看原实现，可以前往Flashbots官网。
- `bsc-rpc-client`:本章节的实验小结部分需要用到的模拟客户端，用于与`bsc-rpc`交互
- `share-node-client`:本章节的实验小结部分需要用到的模拟客户端，用于与`mev-share-node`交互



运行说明：

- 服务端运行见具体仓库文档
- 客户端运行只需要按照`go.mod`文件安装好对应版本的go环境，然后直接`go run main.go`运行接口
